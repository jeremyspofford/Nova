# Nova Dashboard — Ground-Up Redesign

> **Date:** 2026-03-16
> **Status:** Design approved
> **Scope:** Complete visual redesign of Nova's React dashboard (49 existing pages + future pages from roadmap), new backend aggregation endpoints, and a reusable design system.

## Context

Nova's dashboard is functionally complete — 49 pages covering chat, pipeline tasks, goals, pods, models, memory, settings, recovery, RBAC, and onboarding. But it has no strong visual identity: system fonts, generic Tailwind patterns, inconsistent loading/error states, no meaningful motion, and a top-bar navigation that won't scale as the page count grows from ~15 visible nav items to 25+.

The redesign establishes a new design language that communicates **confidence, performance, and privacy** while remaining **approachable for beginners** and **enterprise-capable**. It also accounts for all future pages in the roadmap (Skills, Rules, Devices, Integrations Hub, Event Feed, Browser Viewer, Web IDE, Schedule Editor).

## Approach

**Design System First (Approach A):** Build the complete component library and layout shell before touching any page. Then rebuild every page on the new foundation. This ensures perfect consistency from day one and a reusable system that scales to future pages without ad-hoc decisions.

## Aesthetic Direction

**Confident Modern** — Think Linear meets Vercel. Dark-first palette that signals security and performance. Clean typography with character. Muted teal accent retained from current branding but refined. Progressive disclosure: simple surface, power underneath.

Key principles:
- **Dark-first** — Primary experience is dark mode. Light mode is refined, not an afterthought.
- **Sidebar navigation** — Replaces top bar. Scales to 25+ pages. Categorized sections.
- **Data density with breathing room** — Stat cards, pipeline visualizations, activity feeds. Not cramped.
- **Monospace for technical data** — Task IDs, latencies, costs, timestamps use Geist Mono.
- **Purposeful motion** — Pulse on active tasks, smooth transitions, loading skeletons. No gratuitous animation.
- **Progressive disclosure** — Overview first, details on click/expand. Right-panel sheets for editing.

## Design Decisions & Migration Notes

### Color System Migration

The current codebase uses RGB triplet CSS variables (`--accent-400: 45 212 191`) consumed via Tailwind's `rgb(var(--accent-400) / <alpha-value>)` pattern with a full 50-950 scale per palette and 8+ switchable accent palettes. The redesign introduces **semantic tokens** (`--bg-card`, `--text-primary`, `--accent`) but must preserve the palette-switching capability.

**Strategy: Semantic aliases on top of the existing scale.** Keep the 50-950 palette system in `color-palettes.ts` and `theme-store.tsx`. Add a new layer of semantic CSS variables that reference the scale values. Example:

```css
/* Generated by theme-store based on active palette */
--accent-400: 45 212 191;
--accent-500: 20 184 166;
--neutral-900: 28 25 23;

/* Semantic layer (new) — references the scale */
--bg-root: rgb(var(--neutral-950));
--bg-card: rgb(var(--neutral-900));
--accent: rgb(var(--accent-400));
--accent-dim: rgb(var(--accent-400) / 0.12);
```

This preserves: all existing palette presets, custom accent colors, runtime palette switching, and Tailwind's opacity modifier syntax. The semantic tokens become the primary API for new components; direct scale access (`text-accent-400`) is still available for edge cases.

### Nav Role Changes (Intentional)

The redesign tightens role requirements for several nav items. This is deliberate — the current code is overly permissive:
- Settings: `member` → `admin` (members shouldn't change platform config)
- Keys: `member` → `admin` (API key management is admin)
- Pods: `viewer` → `admin` (agent pipeline config is admin)

### Font Loading

Plus Jakarta Sans: Google Fonts `<link>` in `index.html`.
Geist Mono: Not available on Google Fonts. Install via `npm install geist` and import in CSS: `@import 'geist/font/mono';`. Alternatively, use Fontsource: `npm install @fontsource/geist-mono`.

### Tailwind Version

Stay on Tailwind v3 (current: `^3.4.3`). The semantic CSS custom property approach uses `rgb()` triplets (not hex) to preserve Tailwind v3's opacity modifier support (`bg-accent/10`). See color system migration above.

### New Dependencies

| Package | Purpose |
|---------|---------|
| `geist` or `@fontsource/geist-mono` | Geist Mono font |
| `d3-force` + `d3-selection` (or `@react-sigma/core`) | Force-directed graph for Memory page |

## 2. Component Library

35 components organized by category. Each component supports dark/light mode, responsive sizing, and accessibility (keyboard nav, ARIA labels, focus management).

### 2.1 Primitives

**Button**
- Variants: `primary` (accent bg), `secondary` (subtle bg), `ghost` (no bg), `danger` (red), `outline` (border only)
- Sizes: `sm` (28px h, 12px text), `md` (36px h, 13px text), `lg` (44px h, 14px text)
- States: default, hover, active, disabled, loading (spinner replaces text)
- Icon-only variant with tooltip

**Input**
- Types: text, email, password (reveal toggle), number, url
- Anatomy: optional label (above), input field, optional description/error (below), optional prefix/suffix
- States: default, focused (accent ring), error (red ring), disabled
- Size: single height (36px), consistent with Button md

**Textarea**
- Auto-resize up to max-height
- Optional character count
- Same styling as Input

**Select**
- Native HTML select (styled) for simple cases
- Custom dropdown with search for long option lists (models, tools)
- Same height as Input/Button md

**Checkbox** — Square with checkmark, label + optional description
**Toggle** — Switch with on/off states, label
**Radio** — Radio group with options, descriptions
**Slider** — Range with min/max labels, current value tooltip

### 2.2 Data Display

**Badge**
- Variants: `neutral`, `accent`, `success`, `warning`, `danger`, `info`
- Each has background + text color from the dim/full token pairs
- Sizes: `sm` (20px h, 10px text), `md` (24px h, 11px text)
- Optional dot indicator (left)
- Optional dismiss button (right)

**Avatar**
- Image or initials fallback
- Sizes: `xs` (24px), `sm` (32px), `md` (40px), `lg` (48px)
- Optional status dot (bottom-right corner)
- Group variant for stacked avatars

**StatusDot**
- Colors: `success` (green), `warning` (amber), `danger` (red), `neutral` (gray)
- Optional pulse animation for active/running states
- Sizes: `sm` (6px), `md` (8px), `lg` (10px)

**Code**
- Inline: monospace with subtle background
- Block: dark background, optional syntax highlighting, copy button

**CopyableId**
- Monospace text with truncation
- Click to copy with brief checkmark feedback
- Subtle background pill styling

**Metric**
- Large value (text-display, Geist Mono)
- Label above (text-caption, uppercase, tertiary)
- Optional change indicator below (up/down arrow + percentage, colored)
- Optional sparkline (tiny inline chart)

**ProgressBar**
- Determinate: filled bar with percentage
- Indeterminate: animated shimmer
- Segmented: multiple colored segments (for pipeline stages)

**PipelineStages**
- 5 small rectangular segments in a row
- States per segment: `done` (accent), `active` (accent + pulse), `pending` (border), `failed` (danger)
- Compact variant for table rows, expanded variant for detail views with stage labels

**Table**
- Sortable column headers (click to sort, arrow indicator)
- Selectable rows (checkbox column)
- Expandable rows (chevron to show detail)
- Sticky header on scroll
- Empty state integration
- Responsive: horizontal scroll on small screens

**DataList**
- Key-value pairs in a two-column layout
- Key: text-caption, tertiary. Value: text-body, primary.
- Optional copy button on values
- Used in detail panels

### 2.3 Layout

**Card**
- Base: bg-card, border-subtle, radius-lg
- Variants: `default`, `hoverable` (lift + border change on hover), `selectable` (click state), `outlined` (stronger border)
- Optional header (title + action), optional footer (buttons)
- Glow variant: subtle accent top-border gradient (carried from current design, refined)

**Section**
- Card wrapper for settings/config sections
- Header: icon (accent color) + title + description
- Body: padded content area
- Collapsible variant with chevron toggle

**Modal**
- Sizes: `sm` (400px), `md` (520px), `lg` (680px), `fullscreen`
- Anatomy: backdrop (black 60% opacity), header (title + close button), body (scrollable), footer (action buttons)
- Focus trap, Escape to close, click-outside to close
- Entrance: fade in + scale up (transition-spring)

**Sheet**
- Slide-out right panel for detail views and editing
- Width: 360px (default), 480px (wide), 50vw (half)
- Overlay on mobile, inline on xl+ screens
- Shares Modal's focus trap and close behavior

**Tabs**
- Horizontal tab bar with underline active indicator (accent color)
- Content area below swaps with tab selection
- Scrollable when tabs overflow (horizontal scroll with fade edges)

**Accordion**
- Collapsible sections with chevron indicator
- Smooth height transition
- Single-open or multi-open modes

**EmptyState**
- Centered content: icon (large, tertiary), title, description, action button
- Used in every list/table when no data exists
- Contextual messaging ("No tasks yet — submit your first pipeline task")

**Skeleton**
- Loading placeholders matching actual content shapes
- Rectangular (cards, images), text (multiple lines), circular (avatars)
- Subtle shimmer animation
- Every page has a skeleton variant (not a global spinner)

### 2.4 Navigation

**Sidebar**
- Width: 240px (expanded), 60px (collapsed, icon-only)
- Sections: logo, nav groups (Core, Configure, Monitor, System), user card
- Collapse toggle: chevron button at bottom or hover-triggered
- Active item: accent-dim background + left accent bar
- Badge support on nav items (count, warning)
- Mobile: transforms to bottom tab bar (5 icons) + full-screen drawer

**Breadcrumb**
- For nested pages (Settings > LLM Routing, Pods > Quartet > Task Agent)
- Separator: `/` or `>`
- Last item is current page (not a link)

**CommandPalette**
- Triggered by Cmd+K (Mac) / Ctrl+K (Windows)
- Search input at top, results below grouped by category
- Categories: Pages, Recent Tasks, Goals, Settings, Actions
- Keyboard navigation (arrows + enter)
- Fuzzy search matching

### 2.5 Feedback

**Toast**
- Variants: `success`, `error`, `warning`, `info`
- Position: bottom-right stack
- Auto-dismiss: 5s (success/info), 8s (warning), manual dismiss (error)
- Optional action button ("Undo", "View", etc.)
- Entrance: slide up + fade in

**Tooltip**
- Hover-triggered, 300ms delay
- Arrow pointing to trigger element
- Position: auto (top preferred)
- Dark background in both modes for contrast

**Popover**
- Click-triggered
- Positioned relative to trigger
- Click-outside to close
- Used for menus, filters, quick-actions

**ConfirmDialog**
- Specialized Modal for destructive actions
- Red accent on confirm button
- Type-to-confirm for high-risk actions ("Type DELETE to confirm")

### 2.6 Forms

**SearchInput**
- Input with search icon prefix and clear button suffix
- Debounced onChange (300ms default)
- Optional keyboard shortcut hint ("/")

**ModelPicker**
- Custom Select with model metadata
- Shows: model name, provider icon, context window
- Grouped by provider
- "Auto" option at top with resolved model shown

## 5. Backend Requirements

### 5.1 New Endpoints

All new endpoints are aggregation queries on existing tables. No new data collection infrastructure except the activity events table.

#### 5.1.1 Pipeline Stats
**Endpoint:** `GET /api/v1/pipeline/stats`
**Auth:** Admin or member
**Source:** `tasks` table aggregation
**Response:**
```json
{
  "active_count": 3,
  "queued_count": 1,
  "completed_today": 12,
  "completed_this_week": 47,
  "failed_today": 1,
  "success_rate_7d": 0.94,
  "avg_duration_ms": 23400
}
```
**Query logic:**
- `active_count`: COUNT WHERE status IN ('running', 'context_running', 'task_running', 'guardrail_running', 'review_running')
- `queued_count`: COUNT WHERE status = 'queued'
- `completed_today`: COUNT WHERE status = 'complete' AND completed_at >= today
- `success_rate_7d`: complete / (complete + failed) WHERE completed_at >= 7 days ago
- `avg_duration_ms`: AVG(completed_at - started_at) WHERE status = 'complete' AND completed_at >= 7 days ago

#### 5.1.2 Usage Summary
**Endpoint:** `GET /api/v1/usage/summary?period=day|week|month|year`
**Auth:** Admin
**Source:** `usage_events` table aggregation
**Response:**
```json
{
  "total_cost_usd": 1.24,
  "total_requests": 86,
  "total_input_tokens": 142000,
  "total_output_tokens": 38000,
  "by_model": [{"model": "claude-sonnet-4-6", "cost_usd": 0.82, "requests": 34, "tokens": 89000}],
  "by_day": [{"date": "2026-03-16", "cost_usd": 1.24, "requests": 86}],
  "vs_previous_period_pct": -18.4
}
```
**Query logic:**
- Aggregate usage_events within the requested period
- `vs_previous_period_pct`: compare current period total to the previous equivalent period

#### 5.1.3 Health Overview
**Endpoint:** `GET /api/v1/health/overview`
**Auth:** Admin or member
**Source:** HTTP health check pings to all services with timing
**Implementation:** Orchestrator pings `/health/ready` on each service, measures response time.
**Response:**
```json
{
  "services": [
    {"name": "orchestrator", "status": "up", "latency_ms": 12, "version": "0.1.0"},
    {"name": "llm-gateway", "status": "up", "latency_ms": 8},
    {"name": "memory-service", "status": "up", "latency_ms": 15},
    {"name": "cortex", "status": "up", "latency_ms": 22},
    {"name": "postgres", "status": "up", "latency_ms": 3},
    {"name": "redis", "status": "up", "latency_ms": 1}
  ],
  "avg_latency_ms": 10,
  "overall_status": "healthy"
}
```
**Note:** This is similar to `/api/v1/recovery/status` but with latency timing and from the orchestrator's perspective (not recovery's).

#### 5.1.4 Activity Feed
**Endpoint:** `GET /api/v1/activity?limit=20&offset=0`
**Auth:** Admin or member
**Source:** New `activity_events` table

**New table (migration):**
```sql
CREATE TABLE IF NOT EXISTS activity_events (
    id BIGSERIAL PRIMARY KEY,
    event_type TEXT NOT NULL,       -- task_completed, task_failed, guardrail_finding,
                                     -- goal_progress, config_change, service_event,
                                     -- review_needed, memory_consolidated
    service TEXT NOT NULL,           -- orchestrator, memory-service, cortex, etc.
    severity TEXT NOT NULL DEFAULT 'info',  -- info, warning, error
    summary TEXT NOT NULL,           -- human-readable summary
    metadata JSONB DEFAULT '{}',    -- event-specific data (task_id, goal_id, etc.)
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_activity_events_created ON activity_events(created_at DESC);
CREATE INDEX idx_activity_events_type ON activity_events(event_type);
```

**Producers (INSERT on key events):**
- `orchestrator/app/pipeline/executor.py`: task_completed, task_failed, guardrail_finding, review_needed
- `orchestrator/app/goals_router.py` + goal controller: goal_progress, goal_completed, goal_failed
- `orchestrator/app/router.py` (config PATCH): config_change
- `cortex/app/thinking_loop.py`: cortex_cycle, drive_action
- `memory-service/app/engram/consolidation.py`: memory_consolidated

**Progressive enhancement:** Ship the table + API endpoint + orchestrator producers (pipeline + goals + config) in Phase 4. Add cortex and memory-service producers in a follow-up — the activity feed works without them, just with fewer event types.

**Response:**
```json
[{
  "id": 42,
  "event_type": "guardrail_finding",
  "service": "orchestrator",
  "severity": "warning",
  "summary": "Guardrail Agent flagged credential leak in task-0fa3",
  "metadata": {"task_id": "uuid", "finding_type": "credential_leak", "finding_id": "uuid"},
  "created_at": "2026-03-16T..."
}]
```

#### 5.1.5 Pipeline Latency Stats
**Endpoint:** `GET /api/v1/pipeline/stats/latency`
**Auth:** Admin or member
**Source:** `agent_sessions` table aggregation
**Response:**
```json

  "avg_total_ms": 23400,
  "p50_ms": 18000,
  "p95_ms": 45000,
```

**Query logic:** Aggregate agent_sessions by role, compute avg/p50/p95 from (completed_at - started_at) over last 7 days.
