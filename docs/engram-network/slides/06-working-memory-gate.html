The Context Rot Problem
Every AI today treats the context window as a FIFO queue. Messages go in at the bottom, fall off the top. By message 40:
✗ Early context is summarized to mush
✗ Important decisions forgotten
✗ Model contradicts its own earlier statements
✗ "Lost in the middle" — attention degrades

The Working Memory Gate — Active Context Management
Instead of passively filling a buffer, Nova actively curates what's in the context window every turn. The context window becomes a managed workspace — like a desk where you consciously choose what papers to keep out.

Every Turn — The Gate Cycle
1
Score Existing Items
Every item on the desk gets a relevance score based on: how related to the latest message? Referenced recently? Pinned? Ties to active goal?
2
Activate New Memories
Run spreading activation on the latest message. Reconstruct top activated engrams. These are candidates to enter the desk.
3
Evict & Admit
If desk is full: evict lowest-scoring non-pinned items. Admit new memories that score higher than current lowest. Never silently summarize — either keep or evict.
4
Assemble Prompt
Build the LLM prompt from desk contents: pinned items first (self-model, goal), then memories by relevance, then recent conversation, then the new message.
Why This Kills Context Rot
→ Message 1's important decision? Still on the desk at message 100 — because it's pinned as a key decision, not lost to FIFO
→ Topic shifted? Old context evicts, new context flows in from the Engram Network — desk always matches current thinking
→ Nothing is "lost" — evicted items are still in the graph. They re-activate instantly if the topic returns
→ No summarization mush — items are either present in full or absent. No degraded middle state.
