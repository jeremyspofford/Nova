Request Flow — "User sends a message"
1
User message arrives at orchestrator
↓
2
Orchestrator calls memory-service /engrams/context with message + session state
↓
3
Neural Router (or cosine fallback) produces seed activation weights
↓
4
Spreading activation through engram graph (2-3 hops, <50ms)
↓
5
Reconstruction assembles activated engrams into context-aware narratives
↓
6
Working Memory Gate assembles final prompt: pinned + memories + conversation + message
↓
7
Orchestrator sends assembled prompt to llm-gateway → streams response
↓
8
Response streams to user. Async: raw exchange → Redis ingestion queue → engram decomposition
Total added latency: ~60-100ms (steps 3-6). The LLM call (step 7) is still 500-2000ms. Memory is never the bottleneck.
Migration — Existing Memories → Engrams
1
Schema Migration
New SQL migration creates engrams, engram_edges, engram_archive tables alongside existing tables.
2
Backfill Pipeline
One-time job reads existing semantic/episodic/procedural memories → runs them through decomposition → creates engrams + edges. Reuses existing embeddings.
3
Dual-Write Period
Both old and new systems receive writes. Old endpoints still work (wrapped). Validate engram retrieval against legacy retrieval for quality.
4
Cutover
Once confident, switch orchestrator to engram endpoints. Old tables become read-only archive. No data loss.
Implementation Phases
PHASE 1
Foundation — Engram Storage + Ingestion
Schema, tables, ingestion worker, decomposition prompts. New endpoints. Backfill migration.
Engrams exist. Stored as graph. No new retrieval yet.
PHASE 2
Retrieval — Spreading Activation + Reconstruction
Activation algorithm, reconstruction engine, /engrams/activate + /engrams/reconstruct endpoints.
Nova recalls through the graph for the first time.
PHASE 3
Working Memory — Gate + Prompt Assembly
Working memory slots, gate cycle, /engrams/context endpoint. Orchestrator integration. Self-model bootstrap.
Nova's context window becomes a managed workspace. Context rot dies.
PHASE 4
Consolidation — Sleep Cycle + Self-Model Evolution
Consolidation daemon, pattern extraction, contradiction resolution, pruning, inner loop reflection.
Nova sleeps, dreams, and wakes up wiser.
PHASE 5
Neural Router — Personalized Retrieval
Observation logging, training pipeline, router deployment, phased rollout from shadow mode to full replacement.
Nova's memory becomes uniquely yours.
PHASE 6
Dashboard — Engram Explorer + Self-Model UI
Graph visualization, engram search, self-model inspector, consolidation logs, maturity dashboard.
You can see Nova's mind.
