Neural Memory Router

A small brain that learns YOUR associations — the piece nobody else has built

The Gap Spreading Activation Can't Fill
Spreading activation follows the graph edges that exist. But some of your most important associations aren't in the graph yet. You think "Monday" and your brain jumps to "standup meeting" — not because there's an explicit edge, but because your neural pathways learned that pattern from repetition. The Neural Router learns these implicit associations that the graph hasn't captured.
What It Is
A small, fast neural network (not the LLM) that sits between the query and the Engram Network. It learns to predict which engrams will be useful based on:
INPUT
Query embedding (what you just said)
INPUT
Conversation state (recent topic trajectory)
INPUT
Temporal context (time of day, day of week)
INPUT
Active goal embedding
OUTPUT
Activation weights for engram seeds (replaces/augments cosine similarity)
How It Learns — The Training Loop
1
Observe
Every time Nova retrieves memories, log: the query, the context, the engrams that were surfaced, and which ones the LLM actually used in its response.
2
Label
Positive signal: engrams the LLM referenced. Negative signal: engrams surfaced but ignored. Bonus: user corrections ("no, I meant...") = strong negative.
3
Train
During consolidation, retrain the router on accumulated observations. Contrastive loss: push used engrams up, unused down. Takes seconds on CPU.
4
Deploy
Updated router goes live next session. Seed activation now reflects learned patterns, not just embedding similarity. Gets better every day.
EXAMPLE — AFTER 2 WEEKS OF LEARNING:
Query: "Let's work on the dashboard"
Cosine similarity would seed: dashboard, React, Vite, Tailwind, components...
Neural Router seeds: dashboard + the specific bug Jeremy hit last time + the design pattern Jeremy prefers + the PR feedback from yesterday
The router learned: when Jeremy says "dashboard," he usually wants context on recent issues and his design preferences, not generic React facts.

Model Specs
Architecture: 2-layer MLP + cross-attention
Parameters: ~500K (tiny)
Framework: PyTorch or ONNX
Inference: ~2ms on CPU
Training: seconds during consolidation
Size on disk: ~2MB
Rollout Strategy
Phase 1: Spreading activation only (no router)
Phase 2: Log observations, accumulate training data
Phase 3: Train first router after 200+ observations
Phase 4: Router augments seed activation
Phase 5: Router replaces cosine similarity entirely
Why Not Use the LLM?
Speed: 2ms vs 500ms+ for LLM call
Cost: $0 inference (runs locally)
Personalization: trained on YOUR patterns
Always on: no API dependency
The LLM is the thinker. The router is the librarian.
